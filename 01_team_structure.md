## 연구개발부 조직도

```
연구개발부
├── 백엔드
├── 프론트엔드
├── 앱개발
├── 음성인식 엔진
│   ├── 데이터 구축/가공
│   ├── 모델 연구·훈련
│   ├── 성능검증
│   └── MLOps (모델배포·서빙·모니터링)
```

## 팀별 역할 정의

### 백엔드

| 항목 | 내용 |
|------|------|
| **주요 역할** | 서버 API 개발 및 DB 설계 |
| **책임 범위** | API 개발 · 비즈니스 로직 · DB 관리 |
| **협업 대상** | 프론트엔드 · 앱 · 음성인식 엔진 · DevOps |

| **협업 대상** | 백엔드 · 그래픽팀 · QA |

### 앱개발

| 항목 | 내용 |
|------|------|
| **주요 역할** | iOS · Android 앱 개발 |
| **책임 범위** | 앱 개발 · 스토어 배포 · API 연동 |
| **협업 대상** | 백엔드 · 그래픽팀 · QA |

---

### 음성인식 엔진

| 역할 | 담당 업무 |
|------|-----------|
| **데이터 엔지니어** | 음성 데이터 수집 · 구축 · 가공 |
| **리서처 / ML 엔지니어** | 모델 설계 · 훈련 · 성능검증 |
| **MLOps 엔지니어** | 훈련 인프라 · 모델 배포 · 모니터링 |

**음성인식 엔진 파이프라인**

```
데이터 구축 → 가공 → 모델링(훈련) → 성능검증 → 모델배포
```

**MLOps ↔ DevOps 협업 범위**

```
엔진팀 (MLOps)              DevOps (인프라)
──────────────────           ──────────────────
모델 패키징                  쿠버네티스 / 컨테이너 관리
API 서빙 코드                클라우드 리소스 (GPU 서버 등)
성능 모니터링 정의    ←→    모니터링 인프라 구축
A/B 테스트 설계              트래픽 라우팅 · 스케일링
```

---

### QA

| 항목 | 내용 |
|------|------|
| **주요 역할** | 서비스 품질 검증 |
| **책임 범위** | Staging 기능 검증 · 버그 리포트 · 회귀 테스트 |
| **협업 대상** | 전체 개발팀 |

```
QA 검증 대상
──────────────────────────────
서비스 QA    → 백엔드 · 프론트엔드 · 앱
엔진 성능검증 → 음성인식 엔진팀 내부에서 진행
```

---

### DevOps / 인프라

| 항목 | 내용 |
|------|------|
| **주요 역할** | 개발 인프라 및 배포 파이프라인 관리 |
| **책임 범위** | 클라우드 인프라 · CI/CD · 모니터링 인프라 · Production 배포 승인 |
| **협업 대상** | 전체 개발팀 |

```
DevOps 세부 역할
├── 클라우드 인프라 (GPU 포함)
├── CI/CD 파이프라인 구축 및 관리
├── 모니터링 인프라 구축
└── Production 배포 승인권자
```

### 외부지원

| 팀 | 역할 |
|----|------|
| **그래픽** | UI 디자인 · 시각 자산 제작 (Figma) |
| **기획** | 제품 기획 · 요구사항 정의 |

> ⚠️ 기획팀이 외부지원인 경우, 내부 프로덕트 오너(PO) 역할을 1명 이상 지정하는 것을 권장합니다.
